Vuex

主要用于对复杂应用进行状态管理。集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。 

- 状态：泛指vue组件的一些当前的状况和性质
- 状态管理：对状态进行管理，如a组件这个状态需要通知b组件。（state会显示到View，用户会根据View上的内容进行操作，触发actions，然后再去影响state。）

vuex流程

  • 每次vue组件需要给vuex分派一个actions，actions提交一个mutation，由mutation来修改state，然后再返回给vue组件渲染。
  • state状态只能由mutation来修改
  • actions可以封装各自mutation来进行修改state
  • 总的来说，vuex的状态是存储在store里面的，要修改状态就只能进行mutation的提交commit，一旦进行提交成功，状态被改变后，相关使用该状态的组件也会更新状态信息。

使用vuex的时候怎么用？ 

import Vue from 'vue';
import Vuex from 'vuex';
Vue.use(Vuex);
new Vue.Store({
    state: {  // 放置state的值
        count: 0,
        str: "abcd234"
    },
    getters: {  // 放置getters方法
        strLen: state => state.str.length        
    },
    // mutations只能是同步操作
    mutations: {  // 放置mutations方法
        increment(state, payload){
            state.count = payload.number
        }
    },
    // actions可以是异步操作
    actions: {  // 放置actions方法
        actionName({commit}){
            commit('mutationName')
        }        
    }
});
new Vue({
    el: '#app',
    store,
    ...
})

console.log(Vuex)可以看到，import进来的Vuex实际上是一个对象，里面包含了store构造函数，还有mapActions、mapGetters、mapMutations、mapState辅助方法。

创建好的store实例怎么在各个组件中都能引用

因为store实例是个全局单例，Vuex通过store选项，提供了一种机制将状态从根组件‘注入’到每一个子组件中。

通过在根实例中注册store选项，该store实例会注入到根组件下的所有子组件中，且子组件能通过this.$store访问到。

State
组件所发生的变化，也是直接受到State的驱动来变化的。

通过this.$store.state来直接获取状态，也可以利用vuex提供的mapState辅助函数将state映射到计算属性computed中。

Getters
本质上是用来对状态进行加工处理。getter的返回值会根据它的依赖被缓存起来，且只有当它的依赖值发生了改变才会被重新计算。通过this.$store.getters.valueName对派生出来的状态进行访问。

Mutations
利用mutations可以改变状态。更改vuex的store中的状态的唯一方法就是提交(commit)mutation.但是mutation触发状态改变的方式有些特别，所谓commit一个mutation，实际就像触发一个事件一样，传入一个mutation的类型以及携带一些数据payload载荷。

可以使用this.$store.commit('xxx')提交mutation，还可以使用mapMutation辅助函数将组件中的methods映射为this.$store.commit。

Actions
Action可以包含异步操作 

actions: {
    increment (context) {
      context.commit('increment')
    }
  }

它接收的第一个参数是一个与 store 实例具有相同方法和属性的 context 对象，因此你可以调用 context.commit 提交一个 mutation，或者通过 context.state 和 context.getters 来获取 state 和 getters。

mutation处理函数中所做的事情是改变state，而action处理函数中所做的事情则是commit mutation。

触发action，使用this.$store.dispatch(actionType, payload)

流程上
vue component—-dispatch—->actions—-commit—->mutations—-mutate—->state—-render—->vue component
