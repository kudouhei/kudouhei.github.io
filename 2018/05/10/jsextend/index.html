javascript实现继承的方式

1. 类式继承

function A(){
  this.name = "A";
}
A.prototype.getName = function(name){
  console.log(name)
}

function a(){
  this.name = "a";
}
a.prototype = new A();
var instance = new a();
instance.getName("hello");
console.log(instance.name)

注意：

  • 创建两个类A和a，给A.prototype原型上添加一个共有方法，然后通过new命令实例化一个A，并且赋值给a.prototype原型。
  • 在实例化一个类时，新创建的对象复制了父类的构造函数内的属性与方法并且将原型__proto__指向了父类的原型对象，这样就拥有了父类的原型对象上的属性与方法。
  • 两个缺点：
    • 引用缺陷，新建的不同实例会修改父类中的引用类型
    • 无法为不同的实例初始化继承来的属性。

2. 构造函数继承 

function A(color){
  this.name = 'A';
  this.color = color;
  this.type = ['color1', 'color2'];
}
A.prototype.getName = function(name) {
  console.log(name);
}

function a(color){
  A.apply(this, arguments);
}

var h1 = new a("white"); 
var h2 = new a("blue"); 
console.log(h1.color); // white
h1.type.push("red");  
console.log(h1.type);  // ["color1", "color2", "red"]

console.log(h2.type);  //"color1", "color2"]
console.log(h2.color);  //blue

注意：

  • apply方法的运用，可以改变函数的作用域，所以在a子类中调用这个方法也就是将a子类的变量在父类中执行一遍，这样子类就拥有了父类中的共有属性和方法。
  • 构造函数继承的缺点就是无法获取到父类的共有方法，也就是通过原型prototype绑定的方法。

3. 组合继承
组合继承就是将类式继承和构造函数继承组合在一起。 

function a(color){
  A.apply(this, arguments);
}
a.prototype = new A();

注意：

  • 在子类构造函数中执行父类构造函数，在子类原型上实例化父类。
  • 缺点就是调用了两次父类的构造函数。

4. 寄生组合式继承 

function A(color){
  this.color = color;
}
A.prototype.getName = function(name){
  console.log(name);
}

function a(color){
  A.apply(this,arguments);
}
a.prototype = Object.create(A.prototype);
a.prototype.constructor = a;

var h1 = new a("white");
console.log(h1.color);

注意：

  • 使用Object.create()进行一次浅拷贝，将父类原型上的方法拷贝后赋给a.prototype，这样子类上就能拥有了父类的共有方法，而少一次调用父类的构造函数。

  • Object.create()的浅拷贝作用类似下面函数

    function create(obj){
    function F(){};
    F.prototype = obj;
    return new F();
}

  • 对A的原型进行拷贝后赋给a.prototype，因此a.prototype上的constructor属性也被重写了，所以要进行修复。

5. extends继承
Class和extends是在ES6中新增的，Class用来创建一个类，extends用来实现继承。

class A {
  constructor(color) {
    this.color = color;
  }
  getName(name){
    console.log(name)
  }
}
class a extends A {
  constructor(color){
    super(color);
    this.color = color;
  }
}
let h1 = new a("black");
h1.getName("hello");
console.log(h1.color);

注意：

  • 子类必须在constructor方法中调用super方法，否则新建实例时会报错。这是因为子类没有自己的this对象，而是继承父类的this对象，然后对其进行加工。如果不调用super方法，子类就得不到this对象。
