Vue中methods，computed，watch

methods：方法是能够操作对象的函数，通常是Vue实例本身或者Vue组件。

computed：计算属性对于处理已存在的数据很有价值。当需要对大量数据进行排序并且不想在每次重新计算，可以使用计算属性。

常用的场景例如：

  • 在用户输入时更新大量的信息，例如过滤列表。
  • 从Vuex商店中收集信息
  • 表单验证
  • 数据可视化需要依赖于用户查看的内容而变化

计算属性是基于依赖关系进行缓存的计算，并且在依赖关系变化时进行更新。

watch：侦听数据变化，如果想在数据变化时添加一些功能，或对一些特定的更改做出响应，可以侦听这个属性并应用一些逻辑。

1.计算属性VSmethod方法

  • 计算属性根据它的依赖被缓存，如果message没有被修改，下次get不会进行重复计算
  • method则每次都会重新计算

理解Vue.nextTick

发表于 2018-07-04

理解Vue.nextTick

在下次 DOM 更新循环结束之后执行延迟回调。在修改数据之后立即使用这个方法，获取更新后的 DOM。

原理
Vue实现响应式并不是数据发生变化之后DOM立即变化，而是按一定的策略进行DOM的更新。

Vue在修改数据后，视图不会立刻更新，而是等同一事件循环中的所有数据变化完成之后，在统一进行视图更新。

// 改变数据
vm.message = 'changed'
// 想要立即使用更新后的DOM。这样不行，因为设置message后DOM还没有更新
console.log(vm.$el.textContent) // 不会得到‘changed’
// 这样可以，nextTick里面的代码会在DOM更新后执行
Vue.nextTick(function(){
    console.log(vm.$el.textContent) // 可得到‘changed’
})

只要让nextTick里的代码放在UI render步骤后面执行，就能访问到更新后的DOM了。

vue就是这样的思路，是用队列控制方式得到。

  • vue用异步队列的方式来控制DOM更新和nextTick回调先后执行
  • microtask因为其高优先级特性，能确保队列中的微任务在一次事件循环前被执行完毕

用途
需要在视图更新之后，基于新的视图进行操作。

在created和mounted阶段，如果需要操作渲染后的视图，也要使用nextTick方法

例子： 点击按钮显示原本v-show= false隐藏起来的输入框，并获取焦点 

show() {
    this.showit = true
    this.$nextTick(function(){
        // DOM更新
        document.getElementById("keywords").focus()
    })
}

function formatNumber(str){
  let arr =[],
      count = str.length;
  while(count >= 3){
    arr.unshift(str.slice(count-3, count))
    count -= 3
  }
  str.length % 3 && arr.unshift(str.slice(0, str.length % 3))
  return arr.toString()
}
console.log(formatNumber("123456789"))

function formatNumber1(str){
  return str.replace(/\B(?=(\d{3})+(?!\d))/g, ',')
}
console.log(formatNumber1("999456789")) // 1,234,567,890
  • /B(?=)/,除\B以外的部分都不属于匹配结果，也就是说整个表达式里，真正参与最后的匹配到替换成,的是前面这个\B
  • 先行断言(?=)，如x(?=y)，如果x后面跟的是y，则匹配x。字符串xyz中的x会被匹配，而后面的y是不会被匹配的
  • 后行断言(?!\d),表示后面不能跟相应的内容，这里表示后面不能跟数字了。

常用函数

节流函数

节流函数就是预先设定一个执行周期，当调用动作的时刻大于等于执行周期则执行该动作，然后进入下一个新周期。

适用场景：

  • 适合指定每隔一定时间间隔内执行不超过一次的情况
  • window对象的resize、scroll事件
  • 拖拽时的mousemove事件

实现： 

function throttle(fn, threshhold){
    var timeout;
    var start = new Date;
    var threshhold = threshhold || 160;
    return function(){
        var context = this, args = arguments, curr = new Date() -0;
        clearTimeout(timeout);
        if(curr - start >= threshhold){
            fn.apply(context, args);
            start = curr;
        }else{
            //让方法在脱离事件后也能执行一次
            timeout = setTimeout(function(){
                fn.apply(context, args);
            }, threshhold)
        }
    }
}

去抖函数

去抖函数就是调用动作n毫秒后，才会执行该动作，若在这n毫秒内又调用此动作则将重新计算执行时间。

当持续触发事件时，一定时间段内没有再触发事件，事件处理函数才会执行一次，如果设定的时间到来之前，又一次触发了事件，就重新开始延时。

适用场景：

  • 适合只执行一次的情况。
  • 文字输入、自动完成的keyup事件

实现： 

function debounce(func, delay){
    var timeout;
    return function(e){
        clearTimeout(timeout);
        var context = this, args = arguments;
        timeout = setTimeout(function(){
            func.apply(context, args)
        }, delay)
    }
}

深拷贝实现

基本类型
6种：undefined、null、Boolean、Number、String、Symbol，基本数据类型的值是按值访问的。基本类型的变量是存放在栈内存里。

引用类型
除6种基本数据类型外，剩下的就是引用类型，统称Object类型。细分包括：Object类型、Array类型、Date类型、RegExp类型、Function类型。

  • 引用类型的值是可变的
  • 引用类型的比较是引用的比较
  • 引用类型的值是保存在堆内存中的对象，当需要访问引用类型(对象，数组)的值时，首先从栈中获得该对象的地址指针，然后在从堆内存中取得所需的数据。

javascript存储对象都是存地址的，所以浅拷贝会导致obj1和obj2指向同一块内存地址。改变了其中一方的内容，都是在原来的内存上做修改会导致拷贝对象和源对象都发生改变，而深拷贝是开辟一块新的内存地址，将原对象的各个属性逐个复制进去，对拷贝对象和源对象各自的操作互不影响。

var deepCopy = function(obj){
    if(typeof obj !== 'object')return;
    var newObj = obj instanceof Array ? []: {};
    for(var key in obj){
        if(obj.hasOwnProperty(key){
            newObj[key] = typeof obj[key] === 'object'? deepCopy(obj[key]): obj[key]
        })
    }
    return newObj;
}

js模块化

发表于 2018-06-26

JavaScript模块化

模块化可以使代码低耦合，功能模块不互相影响。模块化的核心思想：

  • 拆分，将js代码按功能逻辑拆分成多个可复用的js代码文件
  • 加载，如何将模块进行加载执行和输出
  • 注入，能够将一个js模块的输出注入到另一个js模块中
  • 依赖管理，管理模块之间的依赖关系

CommonJS & AMD解决模块依赖管理问题，实现不通过全局作用域来实现模块方法。

CommonJS
CommonJS模块可以很方便的将某个对象导出，让他们能够被其他模块通过require语句来引入(Node.js语法)。通过CommonJS，每个js文件独立的存储它模块的内容。在这种作用域中，通过module.exports语句来导出对象为模块，再通过require语句引入。

nodejs的模块系统，就是参照CommonJS规范实现的。

这种模式优点：

  • 避免全局命名空间污染
  • 明确代码之间的依赖关系

但是CommonJS以服务器优先的方式来同步载入模块，假如引入三个模块，会一个个被载入。这种模式在服务端还好，到时在浏览器端，只要它还在读取模块，浏览器载入的页面就会一直卡着不动。

AMD
要想异步加载模块，就要用AMD，异步模块定义规范。AMD规范则是非同步加载模块，允许指定回调函数。由于Node.js主要用于服务器编程，模块文件一般都已经存在于本地硬盘，所以加载起来比较快，不用考虑非同步加载的方式，所以CommonJS规范比较适用。但是，如果是浏览器环境，要从服务器端加载模块，这时就必须采用非同步模式，因此浏览器端一般采用AMD规范。 

define(['myModule', 'myOtherModule'], function(myModule, myOtherModule) {
  console.log(myModule.hello());
});

使用define方法，第一个参数是依赖的模块，这些模块都会在后台无阻塞地加载，第二个参数则作为加载完毕的回调函数。

AMD是优先浏览器的一种异步载入模块的解决方案。AMD的另一个优点是你可以在模块里使用对象、函数、构造函数、字符串、JSON或者别的数据类型，而CommonJS只支持对象。

AMD运行时的核心思想是提前执行依赖。能够带来更好的用户体验，也会产生浪费。

CMD规范
通过按需加载的方式，而不是必须在模块开始就加载所有的依赖。

AMD和CMD的区别：前者是对于依赖的模块提前执行，而后者是延迟执行。 前者推崇依赖前置，而后者推崇依赖就近，即只在需要用到某个模块的时候再require。

ES6模块化(import/export)和Node.js(require基于CommonJS)区别

Node.js无法直接兼容ES6，所有现在require/exports仍然是必须的，目前编写的import/export最终都是会编译为require/exports来执行。

require是CommonJs,AMD,CMD规范中定义的模块请求方式，import是es6规范定义的模块请求方式。

从底层运行来讲，require是在程序运行的时候去解析而import是在编译的时候去做解析请求包，require是请求整个包对象而import是只请求模块中需要的请求的部分。ES6模块的设计思想，是尽量的静态化，使得编译时就能确定模块的依赖关系(“静态优化”)，以及输入和输出的变量。CommonJS和AMD模块，都只能在运行时确定这些东西(“运行时加载”)。

ES6模块加载的机制，与CommonJS模块完全不同。CommonJS模块输出的是一个值的拷贝，而ES6模块输出的是值的引用

ES6 Module与CommonJS对比：

  • CommonJS的模块是对象，而es6的模块不是对象，是编译时执行的，使得编译时就能够确定依赖关系和输入输出的变量。

    // commonJS模块
let {stat, exists, readFile} = require('fs');
//等同于
let _fs = require('fs');
let stat = _fs.stat, exists = _fs.exists, readfile = _fs.readfile;

    上述代码实质是整体加载fs模块，生成一个对象_fs,然后再从这个对象上面读取3个方法。这种加载成为“运行时加载”，因为只有运行时才能得到这个对象，导致完全没办法在编译时做静态优化。

    ES6模块不是对象，而是通过export命令显式指定输出的代码，输入时页采用静态命令的形式。 

    import {stat, exists,readFile} from 'fs'

    上面代码的实质是从fs模块加载3个方法，其他方法不加载。这种加载称为“编译时加载”，即ES6可以在编译时就完成模块加载，效率要比CommonJS模块加载方式高。

  • 模块加载实质不同

    CommonJS加载的模块是对值的拷贝，而es6 Module是对值的引用，这样模块定义文件内部变量的变化会实时反映到依赖文件，而CommonJS不同。 

    // lib.js
export let counter=3;
export function incCounter(){
    counter++;
}
// main.js
import { counter, incCounter } from './lib';
console.log(counter); // 3
incCounter();
console.log(counter); // 4
//模块定义文件的变量counter的变化能够实时反映
export和export default的作用及区别

export和export default实现的功能相同，即：可用于导出常量、函数、文件、模块等，以便其他文件调用。

区别：

  1. export导出多个对象，export default只能导出一个对象
  2. export导出对象需要用{}，export default不要{}
    • export {A,B,C}
    • export default A
  3. 在其他文件引用export default导出的对象时不一定使用导出时的名字。因为这种方式实际上是将该导出对象设置为默认导出对象。

图片懒加载

原理是优先加载在可视区域内的图片，而不一次性加载所有图片。当浏览器滚动，图片进入可视区时再去加载图片。通过设置图片的src属性来让浏览器发起图片请求。当这个属性为空或没有时，就不会发送请求。

判断图片是否进入可视区域
方法一：

  • 通过计算该图片距离document顶部的高度是否小于当前可视区域相对于document顶部高度来判断。

    • document.documentElement.clientHeight获取屏幕可视窗口高度
    • 通过element.offsetTop获取元素相对于文档顶部的距离

    • 通过document.documentElement.scrollTop获取浏览器窗口顶部与文档顶部之间的距离，也就是滚动条滚动的距离。

      var lazyLoad = {
    // 获取元素到页面顶部的距离
    getoffsetTop: function(elem){
        var offTop = 0;
        while(elem!=null){
            offTop += elem.offsetTop; //获取元素到其上一级元素顶部的距离
            elem = elem.offsetParent; //元素上一级元素
        }
        return offTop;
    }
    // 判断是否加载图片
    isLoad: function(elem){
        var cHeight = window.innerHeight || document.body.clientHeight || document.documentElement.clientHeight; //可视窗口的高度
        var sx = window.pageXOffset || document.body.scrollTop || document.documentElement.scrollTop; //浏览器可视窗口距离页面顶部的距离
        var oTop = this.getoffsetTop(elem); //元素到页面顶部的距离
        var viewHeight = oTop-sx; //元素到页面顶部的距离
        // 图片进入浏览器可视区域，加载图片
        if(cHeight >= viewHeight){
            return true;
        }else{
            //图片没进入浏览器可视区域，不加载图片
            return false;
        }
    },
    loadImg: function(elems){
        for(var i=0,len=elems.length; i<len;i++){
            if(this.isLoad(elems[i])){
                if(elems[i].className !== 'loaded'){
                    //针对使用data-自定义的属性，要使用getAttribute()获取值
                    elems[i].src = elems[i].getAttribute('data-url');//实际的url替换原来的url
                    elems[i].className = "loaded"; //被加载的图片，添加class属性
                }
            }else{
                return; //下一个图片没有进入加载区域，不循环
            }
        }
    }
}
//使用
window.onscroll = function(){
    lazyLoad.loadImg(document.getElementsByTagName('img'))
}
//html,src指向为占位符，data-url为图片的实际url

            observer.unobserve(img)
        }
    })
})

function throttle(func, delay, time){
    var timeout, startTime=new Date();
    return function(){
        var context = this,
            args = arguments,
            curTime = new Date();
            clearTimeout(timeout);
        if(curTime - startTime >= time){
            func.apply(context, args);
            startTime = curTime;
            // 没到触发间隔，重新设定定时器
        }else{
            timeout = setTimeout(func, delay)
        }
    }
}

闭包

js 创建函数的同时，创建了函数作用域。本来函数执行完之后，里面的局部变量也就随之注销掉了。然而如果函数返回了另一个函数，那么相当于提供了一个访问那些的局部变量的入口。总之，闭包就是一个函数，能将创建的变量的值始终保持在内存中，以供本地环境使用。

为什么要用闭包

主要涉及JavaScript变量的作用域问题，变量作用域分为全局作用域和局部作用域。

在js中仅有函数function具有作用域，并且函数内部可以访问外部变量，但在函数外部无法访问函数内部变量。简单说，1.使用闭包的主要作用就是间接访问函数的内部数据,2.将创建的变量的值始终保持在内存中，以供本地环境使用。 

function foo(){
    var value = 0;
    return function(){
        console.log(value++);
    }
}
var f = foo()
f();

闭包注意事项
内存泄漏：闭包阻止了垃圾回收机制对变量进行回收，因此变量会永远存在内存中，即使当变量不再被使用时，这样会造成内存泄漏，会严重影响页面的性能。因此当变量对象不再使用时，要将其释放。

闭包应用

  • 模块：闭包能很好的将模块的公有属性、方法暴露出来

    var myModule = (function(window, undefined){
    let name='test';
    function getName(){
        return name;
    }
    return {
        name,
        getName
    }
})(window);
console.log(myModule.name); //test

return关键字将对象引用导出赋值给myModule，从而应用到闭包

  • 存储变量
    • 保存外部函数的变量，原理是基于JavaScript中函数作用域链的特点，内部函数保留了对外部函数的活动变量的引用，所有变量不会被释放。
  • 结果缓存

设想我们有一个处理过程很耗时的函数对象，每次调用都会花费很长时间，那么我们就需要将计算出来的值存储起来，当调用这个函数的时候，首先在缓存中查找，如果找不到，则进行计算，然后更新缓存并返回值，如果找到了，直接返回查找到的值即可。闭包正是可以做到这一点，因为它不会释放外部的引用，从而函数内部的值可以得以保留。

var CachedSearchBox = (function(){
    var cache = {}, count = [];
    return {
        attachSearchBox: function(dsid){
            if(dsid in cache){
                return cache[dsid]
            }
            var fsb = new SearchBox(dsid);//新建
            cache[dsid] = fsb; //更新缓存
            if(count.length>100){
                delete cache[count.shift()]
            }
            return fsb;
        }
    }
})

javascript判断变量类型陷阱

javascript判断变量类型陷阱

  • 六种简单数据类型：String、Number、Boolean、Null、Undefined、Symbol
  • Object是唯一的复杂数据类型。Object、Array、Function类型都可归结为Object复杂数据类型。

typeof：用来检测变量数据类型

对一个值使用typeof可能返回以下结果

  • undefined
  • string
  • boolean
  • number
  • object：值是对象或null,[]
  • function

因为 typeof [] 和 typeof null 都是是会返回 ‘object’的。typeof不能判断数组对象类型。

instanceof：用来测试一个对象在其原型链中是否存在一个构造函数的prototype属性。instanceof不能用来判断五种基本类型值。

用instanceof判断Array类型可以，但是不能安全的判断Object类型。 

[1,2] instanceof Object; //true

用对象的constructor来判断对象类型 

function cstor(variable){
    if (variable === null || variable === undefined) {
        return 'Null or Undefined';
    }
    var cst = variable.constructor;
    switch(cst){
        case Number:
            return 'Number'
        case String:
            return 'String'
        case Boolean:
            return 'Boolean'
        case Array:
            return 'Array'
        case Object:
            return 'Object'
    }
}

用 constructor 判断变量类型有一个致命的缺陷，就是当检测 null 或者 undefined 类型的 constructor 属性时，js会报错。在利用变量的 constructor 属性来判断变量类型时，必须要先保证变量有 不会是 null 或者 undefined。

普适判断变量类型Object.prototype.toString.call()
该方法返回 “[object type]”, 其中type是对象类型。 

Object.prototype.toString.call(null);  //  "[object Null]"
Object.prototype.toString.call([]);  //  "[object Array]"    
Object.prototype.toString.call({});  //  "[object Object]"    
Object.prototype.toString.call(123);  //  "[object Number]"    
Object.prototype.toString.call('123');  //  "[object String]"    
Object.prototype.toString.call(false);  //  "[object Boolean]"    Object.prototype.toString.call(undefined);  //  "[object Undefined]"
