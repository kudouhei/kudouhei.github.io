line-height属性值

发表于 2018-05-01

line-height

行高指文本行基线间的垂直距离。

图中两条红线之间的距离就是行高（line-height），上一行的底线和下一行的顶线之间的距离就是行距，而同一行顶线和底线之间的距离是font-size的大小，行距的一半是半行距. 

半行距 = (line-height - font-size)/2

取值
一般情况下，浏览器默认的line-height为1.2。

值 描述
normal 默认1.2。设置合理的行间距
number 设置数字，此数字会与当前的字体尺寸相乘来设置行间距，即number为当前font-size的倍数
length 设置固定的行间距
% 基于当前字体尺寸的百分比行间距
inherit 规定应该从父元素继承line-height属性的值 
body { font-size:16px; line-height:XXX; }    
h1 {font-size:32px;}    
p {font-size:16px;}

（1）值百分比 

body { font-size: 16px; line-height: 120%;}

line-height的百分比和body的文字大小用来计算，这个计算值会被层叠下去的元素所继承。line-height不会随着相关的font-size做相应的比例缩放。

element font-size line-height 计算后的line-height
body 16px 120% 16*120% =19.2px
h1 32px 继承计算后的值19.2px 19.2px
p 16px 继承计算后的值19.2px 19.2px

（2）值 length 

body { font-size: 16px; line-height:20px}

line-height的20px会被后代继承，所有继承下来的元素会忽略本身的font-size，而使用相同的，继承的line-height。

（3）值 normal
line-height被设置为normal，约为1.2

body { font-size:16px; line-height:normal}

所有继承下来的元素不会忽略本身的font-size，而使用基于font-size算出来的line-height。此时line-height会随着相关的font-size做相应的比例缩放。

element font-size line-height 计算后的line-height
body 16px normal 16*1.2 =19.2px
h1 32px normal 32*1.2=38.4px
p 16px normal 16px*1.2 =19.2px

值：数字
所有继承下来的元素使用基于font-size算出来的line-height。line-height会随着相关的font-size做相应的比例缩放

细说position属性

发表于 2018-04-29

Position属性

属性值有：static、relative、absolute、fixed、sticky

1. static
所有元素默认情况下，position的属性均为static，top/bottom/left/right无效。

2. relative
相对定位。每个元素在页面的普通流中会有“占用”一个位置，这个位置可以理解为默认位置，而相对定位就是将元素偏离元素的默认位置，但普通流中依然保持着原有的默认位置，并没有脱离普通流，只是视觉上发生的偏移。

其他的元素的位置则不会受该元素的影响发生位置改变来弥补它偏离后剩下的空隙。

position：relative没有改变行内元素的display属性

3. absolute
绝对定位。position：absolute的元素会根据最近的定位（position值不是static）的祖先元素。如果绝对定位的元素没有”positioned”祖先元素，那么它是相对于文档的body元素，并且会随着页面滚动而移动。

position：absolute的元素会脱离页面中的普通流而改变display属性

<div class="A">
   A
   <div class="A1">
       A1
       <div class="A2">
           A2
       </div>
   </div>
 </div>


对A2进行绝对定位position:absolute; top:0; left:0;

(1) A,A1不设置

(2) A为position：relative

(3) A为position：absolute

(4) A1为position：absolute

(5) A和A1均为position：absolute

结论

  1. 块状元素在position(relative/static)的情况下width为100%，但是设置了position：absolute之后，会将width变成auto（会受到父元素的宽度影响）。
  2. 元素设置了position：absolute后，如果没有设置top/bottom/left/right属性的话，浏览器会默认设置成auto，而auto的值则是该元素的“默认位置”。

position Tips

（1）应用了position：relative/absolute的元素，margin属性仍然有效。但是会影响精确定位。
（2）position：absolute忽略根元素的padding。

.A1 {
   border: 1px solid red;
   position: relative;
   padding-top: 50px;
 }
 .A2 {
   border: 1px solid green;
   position: absolute;
   top: 0;
   left: 0;
 }


(3) 在IE6/7中设置position属性后会导致z-index属性失效。
解决：在父元素设置一个更大的z-index值即可。

（4）行内元素在应用了position：absolute之后会改变display。

要注意到relative是并没有改变行内元素的呈现模式，而absolute是会改变行内元素的呈现模式，如果设置了absolute并不需要显式的的将元素display改成block。

4. fixed
position：fixed的元素将相对于屏幕视口(viewport)的位置来指定其位置，并且元素的位置在屏幕滚动时不会改变。

  1. 会改变行内元素的呈现模式，使display变更为block。
  2. 会让元素脱离普通流，不占据空间。

失效的position：fixed：
当元素祖先的transform属性非none时，定位容器有视口改为该祖先。

解释：会牵扯到堆叠(stack context)上下文的概念

如何触发元素形成堆叠上下文

  • z-index值不为auto的绝对/相对定位。
  • z-index不为auto的flex项目，即父元素display：flex
  • opacity属性值小于1的元素
  • transform属性值不为none的元素
  • position：fixed
  • will-change中指定了任意CSS属性。

5. sticky属性
粘性定位。结合了position：relative和position：fixed两种定位功能于一体的特殊定位。

{
    position: -webkit-sticky;
    position: sticky;
    top: 20px;
}

因为设定的阈值是 top:20px ，这个值表示当元素距离页面视口（Viewport，也就是fixed定位的参照）顶部距离大于 20px 时，元素以 relative 定位表现，而当元素距离页面视口小于 20px 时，元素表现为 fixed 定位，也就会固定在顶部。

理解float & display：inline-block

发表于 2018-04-28

理解分析float

float即为浮动，使用float，该浮动元素便会脱离正常的文档流，使其浮动到父元素的行的一端（元素向右或向左移动，直到碰到浮动元素，或者达到父元素内容的边界），并且其他元素将忽略该浮动元素并填补其原先的空间。

文档流：元素按照自上向下，自左到右的顺序依次排放。

float特性

1.包裹性
block元素不指定width的话，默认是100%，一旦让该div浮动起来，就会产生类似inline元素的包裹性，宽度会跟随内容自适应。（通常float元素需要手动指定width）

浮动能设定为左浮和右浮，但display:inline-block都是从左到右排列的。

对比 inline-block
inline-block的元素是宽高可设置，相邻的元素会在一行显示，直到本行排满，也就是让元素的容器属性为block，内容为inline。

  1. inline-block: 水平排列一行，即使元素高度不一，也会以高度最大的元素高度为行高，即使高度小的元素周围留空，也不回有第二行元素上浮补位。可以设置默认的垂直对齐基线。inline-block的元素仍然在仍在文档流中占据位置。
  2. float: 让元素脱离当前文档流，呈环绕状排列，如遇上行有空白，而当前元素大小可以挤进去，这个元素会在上行补位排列。默认是顶部对齐。脱离文档流。
  3. 当我们要设置某些元素在一行显示，并且排列方向一致的情况下，我们尽可能去用inline-block。
    因为inline-block的元素仍然在仍在文档流中占据位置。，这样就减少了程序对DOM的更改操作，因为DOM的每一次更改，浏览器会重绘DOM树。会增加性能消耗。这样也不用像float那样麻烦，需要清除flaot.

2.高度欺骗 

<div style="border:4px solid red;">
    <img style="border:4px solid yellow;float:left;" src="img/25/2.jpg" />
</div>

外层div没有了浮动，因此红色边框宽度默认是100%全屏。其内容img由于加上了float，使得该img具有了欺骗性。float给img施了个障眼法，让该img的inline-height高度塌陷为0了。这样外层div计算高度时，认为img的高度为0，相当于div的content的高度为0，因此红色边框看起来是一条直线。

js实现数组去除重复项

发表于 2017-11-22

js去除数组重复项

方法1：js对象的特性：在js对象中key是永远不会重复的

  1. 把数组转成一个js的对象
  2. 把数组中的值，变成js对象当中的key
  3. 把这个对象，在还原成数组

代码 

function unique(arr){
   var obj = {};
   var _arr = [];
   for(var i = 0,len = arr.length;i<len;i++){
     obj[arr[i]] = true;
   } 

   for(var attr in obj){
     if(obj.hasOwnProperty(attr)){
       _arr.push(attr);
     }
   }
   return _arr.map(function(item){ return +item});
}  
var arr=[1,1,2,3,3,4,5,6];
console.log(unique(arr));

方法2：遍历数组法： 这应该是最简单的去重方法（实现思路：新建一新数组，遍历数组，值不在新数组就加入该新数组中） 

// 遍历数组去重法
function unique(arr){
  var _arr = []
  //遍历当前数组
  for(var i = 0; i < arr.length; i++){
    //如果当前数组的第i已经保存进了临时数组，那么跳过，
    //否则把当前项push到临时数组里面
    if (_arr.indexOf(arr[i]) == -1) _arr.push(arr[i])
  }
  return _arr
}

方法3：对象键值对。速度快，高效，占用更大的内存换取更快的时间，用 JavaScript 中的 Object 对象来当做哈希表，hash去重的核心是构建了一个 hash 对象来替代 indexOf 

 // hash 去重法
function unique(arr){
    var _arr = [], 
        hash = {}
    for (var i = 0; i < arr.length; i++) {
        var item = arr[i]
        var key = typeof(item) + item
          // 对象的键值只能是字符串, typeof(item) + item来去分1和'1'的情况
        if(hash[key] !== 1){
            _arr.push(item)
            hash[key] = 1
        }
    }
    return _arr
}

方法4：es6 Set数据结构：ES6 提供了新的数据结构 Set。它类似于数组，但是成员的值都是唯一的，没有重复的值 

function unique(arr){
    return Array.from(new Set(arr))
    // Array.from方法用于将两类对象转为真正的数组：
    // 类似数组的对象（array-like object）和可遍历（iterable）的对象
}

方法5：Map实现 

function unique(arr){
    const newArray = [];
    const tmp = new Map();
    for(let i=0;i<arr.length;i++){
        if(!tmp.get(arr[i])){
            tmp.set(arr[i],1);
            newArray.push(arr[i]);
        }
    }
    return newArray;
}

绝对底部sticky_footers

发表于 2017-11-20 | 分类于 CSS

绝对底部 CSS sticky footers

如果页面内容不够长的时候，页脚块粘贴在视窗底部；如果内容足够长时，页脚块会被内容向下推送。

方法1-经典套路

给内容区域设置min-height:100%，将footer推到屏幕下方然后给footer添加margin-top，其值footer高度的负值，就能让footer回到屏幕底部。 

<div class="wrap">
  <div class="content">
    <p>填充内容</p>
  </div>
</div>
<div class="footer">
  <p>这是页脚</p>
</div>
html,body {
  height: 100%;
}

body > .wrap {
  min-height: 100%;
}

.content {
  /* padding-bottom 等于 footer 的高度 */
  padding-bottom: 60px;
}

.footer {
  width: 100%;
  height: 60px;
  /* margin-top 为 footer 高度的负值 */
  margin-top: -60px;
}

说明如果主体是使用的悬浮布局，还得解决一些浏览器的兼容性问题，清除浮动 

.clearfix{
    display: inline-block;
}
.clearfix :after{
    content:'.';
    display: block;
    height: 0;
    line-height: 0;
    clear: both;
    visibility: hidden;
}
