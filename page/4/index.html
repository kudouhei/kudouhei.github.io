发表于 2018-05-19

前端路由

  • 在页面不刷新的情况下实现url的变化
  • 捕捉url的变化，根据url更改页面内容

前端路由和服务端路由的区别

  • 服务端路由：每次访问一个新页面的时候都要向服务器发送请求，然后服务器再响应请求，这个过程肯定会有延迟
  • 前端路由在访问一个新页面的时候仅仅是变换了一下路径而已，没有了网络延迟，对于用户体验来说会有相当大的提升

1. hash路由
hash路由明显的标志是带有#，主要是通过监听url中的hash变化来进行路由跳转。hash值变化，并不会导致浏览器向服务器发出请求，浏览器不发出请求，也就不会刷新页面。每次hash值的变化，还会触发hashchange事件，通过这个事件就可以知道hash值发生了哪些变化。优势就是兼容性好

class Routers {
          constructor(){
              // 以键值对的形式储存路由
              this.routes = {};
              // 当前路由的url
              this.currentUrl = '';
              // 记录出现过的hash
              this.history = [];
              // 作为指针，默认指向this.history的末尾，根据后退前进指向history中不同的hash
              this.currentIndex = this.history.length-1;
              this.refresh = this.refresh.bind(this);
              this.back = this.back.bind(this);
              // 默认不是后退
              this.isBack = false;
              //监听对应事件
              window.addEventListener('load',this.refresh, false);
              window.addEventListener('haschange', this.refresh, false);
          }
          // 将path路径与对应的callback函数存储
          route(path, callback){
              this.routes[path] = callback || function(){}
          }
          // 刷新
          refresh(){
              // 获取当前url中的hash路径
              this.currentUrl = location.hash.slice(1) || '/'
              if(!this.isBack){
                  // 如果不是后退操作,且当前指针小于数组总长度,直接截取指针之前的部分储存下来
                  if(this.currentIndex<this.history.length-1)
                    this.history = this.history.slice(0, this.currentIndex+1);
                  // 将当前hash路由推入数组储存
                  this.history.push(this.currentUrl);
                  this.currentIndex++;
              }
              // 执行当前hash路径的callback函数
              this.routes[this.currentUrl]();
              this.isBack = false;
          }
          back(){
              this.isBack = true;
              this.currentIndex <= 0 ? (this.currentIndex=0):(this.currentIndex=this.currentIndex-1);
              location.hash = `#${this.history[this.currentIndex]}`;
              this.routes[this.history[this.currentIndex]]();
          }
      }

2. History API
history.pushState用于在浏览器历史中添加历史记录，但是并不触发跳转，此方法接受三个参数

  • state：与指定网址相关的状态对象，popstate事件触发时，该对象会传入回调函数
  • title：新页面标题
  • url：新的网址，必须与当前页面处在同一个域

popstate事件，每当同一个文档的浏览历史（History对象）出现变化时，就会触发popstate事件。

class Routers {
  constructor() {
    this.routes = {};
    // 在初始化时监听popstate事件
    this._bindPopState();
  }
  // 初始化路由
  init(path) {
    history.replaceState({path: path}, null, path);
    this.routes[path] && this.routes[path]();
  }
  // 将路径和对应回调函数加入hashMap储存
  route(path, callback) {
    this.routes[path] = callback || function() {};
  }

  // 触发路由对应回调
  go(path) {
    history.pushState({path: path}, null, path);
    this.routes[path] && this.routes[path]();
  }
  // 监听popstate事件
  _bindPopState() {
    window.addEventListener('popstate', e => {
      const path = e.state && e.state.path;
      this.routes[path] && this.routes[path]();
    });
  }
}

javascript实现继承的方式

发表于 2018-05-10

javascript实现继承的方式

1. 类式继承

function A(){
  this.name = "A";
}
A.prototype.getName = function(name){
  console.log(name)
}

function a(){
  this.name = "a";
}
a.prototype = new A();
var instance = new a();
instance.getName("hello");
console.log(instance.name)

注意：

  • 创建两个类A和a，给A.prototype原型上添加一个共有方法，然后通过new命令实例化一个A，并且赋值给a.prototype原型。
  • 在实例化一个类时，新创建的对象复制了父类的构造函数内的属性与方法并且将原型__proto__指向了父类的原型对象，这样就拥有了父类的原型对象上的属性与方法。
  • 两个缺点：
    • 引用缺陷，新建的不同实例会修改父类中的引用类型
    • 无法为不同的实例初始化继承来的属性。

2. 构造函数继承 

function A(color){
  this.name = 'A';
  this.color = color;
  this.type = ['color1', 'color2'];
}
A.prototype.getName = function(name) {
  console.log(name);
}

function a(color){
  A.apply(this, arguments);
}

var h1 = new a("white"); 
var h2 = new a("blue"); 
console.log(h1.color); // white
h1.type.push("red");  
console.log(h1.type);  // ["color1", "color2", "red"]

console.log(h2.type);  //"color1", "color2"]
console.log(h2.color);  //blue

注意：

  • apply方法的运用，可以改变函数的作用域，所以在a子类中调用这个方法也就是将a子类的变量在父类中执行一遍，这样子类就拥有了父类中的共有属性和方法。
  • 构造函数继承的缺点就是无法获取到父类的共有方法，也就是通过原型prototype绑定的方法。

3. 组合继承
组合继承就是将类式继承和构造函数继承组合在一起。 

function a(color){
  A.apply(this, arguments);
}
a.prototype = new A();

注意：

  • 在子类构造函数中执行父类构造函数，在子类原型上实例化父类。
  • 缺点就是调用了两次父类的构造函数。

4. 寄生组合式继承 

function A(color){
  this.color = color;
}
A.prototype.getName = function(name){
  console.log(name);
}

function a(color){
  A.apply(this,arguments);
}
a.prototype = Object.create(A.prototype);
a.prototype.constructor = a;

var h1 = new a("white");
console.log(h1.color);

注意：

  • 使用Object.create()进行一次浅拷贝，将父类原型上的方法拷贝后赋给a.prototype，这样子类上就能拥有了父类的共有方法，而少一次调用父类的构造函数。

  • Object.create()的浅拷贝作用类似下面函数

    function create(obj){
    function F(){};
    F.prototype = obj;
    return new F();
}

  • 对A的原型进行拷贝后赋给a.prototype，因此a.prototype上的constructor属性也被重写了，所以要进行修复。

5. extends继承
Class和extends是在ES6中新增的，Class用来创建一个类，extends用来实现继承。

class A {
  constructor(color) {
    this.color = color;
  }
  getName(name){
    console.log(name)
  }
}
class a extends A {
  constructor(color){
    super(color);
    this.color = color;
  }
}
let h1 = new a("black");
h1.getName("hello");
console.log(h1.color);

注意：

  • 子类必须在constructor方法中调用super方法，否则新建实例时会报错。这是因为子类没有自己的this对象，而是继承父类的this对象，然后对其进行加工。如果不调用super方法，子类就得不到this对象。

instanceof和typeof实现原理

typeof被用来判断一个变量的类型，可以使用typeof来判断number，string，object，boolean，function，undefined，symbol七种类型。typeof在判断不是object类型的数据的时候，能清楚的告诉我们是哪一类的类型。但是在判断一个object的数据的时候只能告诉我们这个数据是object，不能细致到具体哪一种。要判断数据具体是哪一种object，需要利用instanceof

typeof原理

js在底层存储变量的时候，会在变量的机器码的低位1-3位存储其类型信息。

  • 000：对象
  • 010：浮点数
  • 100：字符串
  • 110：布尔
  • 1：整数

对于undefined和null，这两个值的信息存储是优点特殊的。

  • null: 所有机器码均为0
  • undefined：用-2^30整数来表示

所以typeof在判断null的时候，由于所有的机器码均为0，因此直接被当做了对象来看待。所以在用typeof来判断变量类型的时候，最好用typeof来判断基本数据类型，避免对null的判断

可以用Object.prototype.toString.call()对变量的类型进行比较准确的判断

instanceof实现原理

instanceof主要作用就是判断一个实例是否属于某种类型。也可以判断一个实例是否是其父类型或祖先类型的实例。

function new_instance_of(leftValue, rightValue){
    let rightProto = rightValue.prototype;
    let leftValue = leftValue.__proto__;
    while(true){
        if(leftValue === null){
            return false;
        }
        if(leftValue === rightProto){
            return true;
        }
        leftValue = leftValue.__proto__
    }
}

主要实现原理就是只要右边变量的prototype在左边变量的原型链上即可。

null和undefined区别

null是一个表示‘无’的对象，转为数值时为0；undefined是一个表示‘无’的原始值，转为数值时为NaN。

undefined：

  • 变量被声明了，但没有赋值时，就等于undefined
  • 调用函数时，应该提供的参数没有提供，该参数等于undefined
  • 对象没有赋值的属性，该属性值为undefined
  • 函数没有返回值时，默认返回undefined

null：

  • 作为函数的参数，表示该函数的参数不是对象
  • 作为对象原型链的终点

如何理解js作用域链

当执行一段js代码时，js引擎会为其创建一个作用域又称为执行上下文，在页面加载后会首先创建一个全局的作用域，然后每执行一个函数，会建立一个对应的作用域，从而形成了一条作用域链。每个作用域都有一条对应的作用域链，链头是全局作用域，链尾是当前函数作用域。

作用域链的作用是用于解析标识符，当函数被创建时（不是执行），会将this，arguments，命名参数和该函数中的所有局部变量添加到该当前作用域中，当JavaScript需要查找变量x的时候（这过程为变量解析），它首先会从作用域链中的链尾也就是当前作用域进行查找是否有x属性，如果没有找到就顺着作用域链继续查找，直到查找到链头，也就是全局作用域链，仍未找到该变量的话，就认为这段代码的作用域链上不存在x变量，并抛出一个引用错误的异常。

如何理解js的原型链

  • 每一个函数对象都有一个prototype属性，该属性指向其prototype对象。

  • 每一个对象都有一个__proto__属性，该属性指向创建这个对象的构造函数的prototype对象。

    function Person(){}
var person = new Person();
person.__proto__ == Person.prototype  // true
Person.prototype.__proto__ === Object.prototype // true
Object.prototype.__proto__ == null

Person.prototype.constructor == Person // true
//获得对象的原型
Object.getPrototypeOf(person) === Person.prototype // true
person.prototype  // undefined
person.__proto__.__proto__ === Object.prototype
person.constructor // function Person(){}
Person.__proto__ === Function.prototype  // true

相互关联的原型组成的链状结构就是原型链。当读取实例的属性时，如果找不到，就会查找与对象关联的原型中的属性，如果还查不到，就去找原型的原型，一直找到最顶层Object.prototype为止。

原型链的作用用于对象继承，函数A的原型属性是一个对象，当这个函数被用作构造函数来创建实例时，该函数的原型属性将被作为原型赋值给所有对象实例，比如新建一个数组，数组的方法便从数组原型上继承而来。

ES5中的prototype与ES6中class继承的比较
ES5继承：

  • 通过原型链实现继承。子类的prototype为父类对象的一个实例，因此子类的原型对象包含指向父类的原型对象的指针，父类的实例属性成为子类原型属性。
  • ES5的继承，实质是先创造子类的实例对象this，然后再将父类的方法添加到this上面。

ES6继承：

  • 子类没有自己的this对象，因此必须在constructor中通过super继承父类的this对象，而后对此this对象进行加工。super关键字在构造函数中表示父类的构造函数，用来新建父类的this对象。
  • ES6的继承机制完全不同，实质是先创造父类的实例对象this（必须先调用super方法），然后在用子类的构造函数修改this。
  • super可作为函数和对象使用。当作为函数使用时，只可在子类的构造函数中使用，表示父类的构造函数，但是super中的this指向的是子类的实例，因此在子类中super()表示的是Parent.prototype.consturctor.call(this)。当作为对象使用时，super表示父类原型对象，即Parent.prototype

箭头函数this

箭头函数this

箭头函数不会创造独立的上下文，它的上下文来自其所在上下文的this作为自己的this。

var person = {
  name: "test",
  getName: () => console.log(this.name)
}
person.getName() // undefined

事实上，箭头函数的this是其所在上下文的this决定的。这里的箭头函数()=>console.log(this.name)的上下文是全局上下文也就是window。person.getName()调用getName函数时，this指向全局上下文，而此时全局上下文中没有name（因为箭头函数没有自己的this上下文，那么当它作为对象的方法函数运行时，this并不指向这个对象）

var person = {
  name: "test",
  getName(){
    return () => console.log(this.name)
  }
}
person.getName()(); // test

此时，()=>console.log(this.name)中的this是由getName()的上下文的this决定的，当调用person.getName()时，getName()的上下文是person对象，因此箭头函数的上下文为person对象。所以person.getName()()虽然执行时的上下文是window，但是箭头函数的this却是指向person对象。

function Person(){
  this.count = 0;
  setInterval(()=> {
    console.log(this.count++);
  },1000)
}
var p = new Person() // 0 1 2...

如果定时器内部的函数是普通函数，那么函数的上下文是全局window，改为箭头函数之后，()=>{this.count++}的this是由所在的上下文决定的，当执行 var p = new Person()时，Person函数内部的上下文的this便指向p，因此箭头函数内部的上下文也必定是p，this.count++可正常执行。

this对象的理解

this是js的一个关键字，随着函数使用场合不同，this的值会发生变化。但是有一个总原则，this是运行时绑定的，总是指向函数的直接调用者。

作为方法调用时，this就是这这个对象。如果有new关键字，this指向new出来的那个对象；在事件中，this指向触发这个事件的对象，但是IE中的attachEvent中的this总是指向全局对象Window

箭头函数与普通函数区别

  • 箭头函数没有this，所以需要通过查找作用域链来确定this的值。
  • 箭头函数没有自己的arguments对象，因为箭头函数可以访问外围函数的arguments对象。
  • 不能通过new关键字调用
  • 没有new.target
  • 没有原型，由于不能使用new调用箭头函数，所有没有构建原型的需求。
  • 没有super，由于没有原型，所有不能通过super来访问原型的属性，所有箭头函数也没有super

line-height属性值

发表于 2018-05-01

line-height

行高指文本行基线间的垂直距离。

图中两条红线之间的距离就是行高（line-height），上一行的底线和下一行的顶线之间的距离就是行距，而同一行顶线和底线之间的距离是font-size的大小，行距的一半是半行距. 

半行距 = (line-height - font-size)/2

取值
一般情况下，浏览器默认的line-height为1.2。

值 描述
normal 默认1.2。设置合理的行间距
number 设置数字，此数字会与当前的字体尺寸相乘来设置行间距，即number为当前font-size的倍数
length 设置固定的行间距
% 基于当前字体尺寸的百分比行间距
inherit 规定应该从父元素继承line-height属性的值 
body { font-size:16px; line-height:XXX; }    
h1 {font-size:32px;}    
p {font-size:16px;}

（1）值百分比 

body { font-size: 16px; line-height: 120%;}

line-height的百分比和body的文字大小用来计算，这个计算值会被层叠下去的元素所继承。line-height不会随着相关的font-size做相应的比例缩放。

element font-size line-height 计算后的line-height
body 16px 120% 16*120% =19.2px
h1 32px 继承计算后的值19.2px 19.2px
p 16px 继承计算后的值19.2px 19.2px

（2）值 length 

body { font-size: 16px; line-height:20px}

line-height的20px会被后代继承，所有继承下来的元素会忽略本身的font-size，而使用相同的，继承的line-height。

（3）值 normal
line-height被设置为normal，约为1.2

body { font-size:16px; line-height:normal}

所有继承下来的元素不会忽略本身的font-size，而使用基于font-size算出来的line-height。此时line-height会随着相关的font-size做相应的比例缩放。

element font-size line-height 计算后的line-height
body 16px normal 16*1.2 =19.2px
h1 32px normal 32*1.2=38.4px
p 16px normal 16px*1.2 =19.2px

值：数字
所有继承下来的元素使用基于font-size算出来的line-height。line-height会随着相关的font-size做相应的比例缩放

细说position属性

发表于 2018-04-29

Position属性

属性值有：static、relative、absolute、fixed、sticky

1. static
所有元素默认情况下，position的属性均为static，top/bottom/left/right无效。

2. relative
相对定位。每个元素在页面的普通流中会有“占用”一个位置，这个位置可以理解为默认位置，而相对定位就是将元素偏离元素的默认位置，但普通流中依然保持着原有的默认位置，并没有脱离普通流，只是视觉上发生的偏移。

其他的元素的位置则不会受该元素的影响发生位置改变来弥补它偏离后剩下的空隙。

position：relative没有改变行内元素的display属性

3. absolute
绝对定位。position：absolute的元素会根据最近的定位（position值不是static）的祖先元素。如果绝对定位的元素没有”positioned”祖先元素，那么它是相对于文档的body元素，并且会随着页面滚动而移动。

position：absolute的元素会脱离页面中的普通流而改变display属性

<div class="A">
   A
   <div class="A1">
       A1
       <div class="A2">
           A2
       </div>
   </div>
 </div>


对A2进行绝对定位position:absolute; top:0; left:0;

(1) A,A1不设置

(2) A为position：relative

(3) A为position：absolute

(4) A1为position：absolute

(5) A和A1均为position：absolute

结论

  1. 块状元素在position(relative/static)的情况下width为100%，但是设置了position：absolute之后，会将width变成auto（会受到父元素的宽度影响）。
  2. 元素设置了position：absolute后，如果没有设置top/bottom/left/right属性的话，浏览器会默认设置成auto，而auto的值则是该元素的“默认位置”。

position Tips

（1）应用了position：relative/absolute的元素，margin属性仍然有效。但是会影响精确定位。
（2）position：absolute忽略根元素的padding。

.A1 {
   border: 1px solid red;
   position: relative;
   padding-top: 50px;
 }
 .A2 {
   border: 1px solid green;
   position: absolute;
   top: 0;
   left: 0;
 }


(3) 在IE6/7中设置position属性后会导致z-index属性失效。
解决：在父元素设置一个更大的z-index值即可。

（4）行内元素在应用了position：absolute之后会改变display。

要注意到relative是并没有改变行内元素的呈现模式，而absolute是会改变行内元素的呈现模式，如果设置了absolute并不需要显式的的将元素display改成block。

4. fixed
position：fixed的元素将相对于屏幕视口(viewport)的位置来指定其位置，并且元素的位置在屏幕滚动时不会改变。

  1. 会改变行内元素的呈现模式，使display变更为block。
  2. 会让元素脱离普通流，不占据空间。

失效的position：fixed：
当元素祖先的transform属性非none时，定位容器有视口改为该祖先。

解释：会牵扯到堆叠(stack context)上下文的概念

如何触发元素形成堆叠上下文

  • z-index值不为auto的绝对/相对定位。
  • z-index不为auto的flex项目，即父元素display：flex
  • opacity属性值小于1的元素
  • transform属性值不为none的元素
  • position：fixed
  • will-change中指定了任意CSS属性。

5. sticky属性
粘性定位。结合了position：relative和position：fixed两种定位功能于一体的特殊定位。

{
    position: -webkit-sticky;
    position: sticky;
    top: 20px;
}

因为设定的阈值是 top:20px ，这个值表示当元素距离页面视口（Viewport，也就是fixed定位的参照）顶部距离大于 20px 时，元素以 relative 定位表现，而当元素距离页面视口小于 20px 时，元素表现为 fixed 定位，也就会固定在顶部。

理解float & display：inline-block

发表于 2018-04-28

理解分析float

float即为浮动，使用float，该浮动元素便会脱离正常的文档流，使其浮动到父元素的行的一端（元素向右或向左移动，直到碰到浮动元素，或者达到父元素内容的边界），并且其他元素将忽略该浮动元素并填补其原先的空间。

文档流：元素按照自上向下，自左到右的顺序依次排放。

float特性

1.包裹性
block元素不指定width的话，默认是100%，一旦让该div浮动起来，就会产生类似inline元素的包裹性，宽度会跟随内容自适应。（通常float元素需要手动指定width）

浮动能设定为左浮和右浮，但display:inline-block都是从左到右排列的。

对比 inline-block
inline-block的元素是宽高可设置，相邻的元素会在一行显示，直到本行排满，也就是让元素的容器属性为block，内容为inline。

  1. inline-block: 水平排列一行，即使元素高度不一，也会以高度最大的元素高度为行高，即使高度小的元素周围留空，也不回有第二行元素上浮补位。可以设置默认的垂直对齐基线。inline-block的元素仍然在仍在文档流中占据位置。
  2. float: 让元素脱离当前文档流，呈环绕状排列，如遇上行有空白，而当前元素大小可以挤进去，这个元素会在上行补位排列。默认是顶部对齐。脱离文档流。
  3. 当我们要设置某些元素在一行显示，并且排列方向一致的情况下，我们尽可能去用inline-block。
    因为inline-block的元素仍然在仍在文档流中占据位置。，这样就减少了程序对DOM的更改操作，因为DOM的每一次更改，浏览器会重绘DOM树。会增加性能消耗。这样也不用像float那样麻烦，需要清除flaot.

2.高度欺骗 

<div style="border:4px solid red;">
    <img style="border:4px solid yellow;float:left;" src="img/25/2.jpg" />
</div>

外层div没有了浮动，因此红色边框宽度默认是100%全屏。其内容img由于加上了float，使得该img具有了欺骗性。float给img施了个障眼法，让该img的inline-height高度塌陷为0了。这样外层div计算高度时，认为img的高度为0，相当于div的content的高度为0，因此红色边框看起来是一条直线。

js实现数组去除重复项

js去除数组重复项

方法1：js对象的特性：在js对象中key是永远不会重复的

  1. 把数组转成一个js的对象
  2. 把数组中的值，变成js对象当中的key
  3. 把这个对象，在还原成数组

代码 

function unique(arr){
   var obj = {};
   var _arr = [];
   for(var i = 0,len = arr.length;i<len;i++){
     obj[arr[i]] = true;
   } 

   for(var attr in obj){
     if(obj.hasOwnProperty(attr)){
       _arr.push(attr);
     }
   }
   return _arr.map(function(item){ return +item});
}  
var arr=[1,1,2,3,3,4,5,6];
console.log(unique(arr));

方法2：遍历数组法： 这应该是最简单的去重方法（实现思路：新建一新数组，遍历数组，值不在新数组就加入该新数组中） 

// 遍历数组去重法
function unique(arr){
  var _arr = []
  //遍历当前数组
  for(var i = 0; i < arr.length; i++){
    //如果当前数组的第i已经保存进了临时数组，那么跳过，
    //否则把当前项push到临时数组里面
    if (_arr.indexOf(arr[i]) == -1) _arr.push(arr[i])
  }
  return _arr
}

方法3：对象键值对。速度快，高效，占用更大的内存换取更快的时间，用 JavaScript 中的 Object 对象来当做哈希表，hash去重的核心是构建了一个 hash 对象来替代 indexOf 

 // hash 去重法
function unique(arr){
    var _arr = [], 
        hash = {}
    for (var i = 0; i < arr.length; i++) {
        var item = arr[i]
        var key = typeof(item) + item
          // 对象的键值只能是字符串, typeof(item) + item来去分1和'1'的情况
        if(hash[key] !== 1){
            _arr.push(item)
            hash[key] = 1
        }
    }
    return _arr
}

方法4：es6 Set数据结构：ES6 提供了新的数据结构 Set。它类似于数组，但是成员的值都是唯一的，没有重复的值 

function unique(arr){
    return Array.from(new Set(arr))
    // Array.from方法用于将两类对象转为真正的数组：
    // 类似数组的对象（array-like object）和可遍历（iterable）的对象
}

方法5：Map实现 

function unique(arr){
    const newArray = [];
    const tmp = new Map();
    for(let i=0;i<arr.length;i++){
        if(!tmp.get(arr[i])){
            tmp.set(arr[i],1);
            newArray.push(arr[i]);
        }
    }
    return newArray;
}

给内容区域设置min-height:100%，将footer推到屏幕下方然后给footer添加margin-top，其值footer高度的负值，就能让footer回到屏幕底部。

  height: 100%;
}

  min-height: 100%;
}

  padding-bottom: 60px;
}

.footer {
  width: 100%;
  height: 60px;
  /* margin-top 为 footer 高度的负值 */
  margin-top: -60px;
}
